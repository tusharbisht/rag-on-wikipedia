# RAG-on-Wikipedia Service
_Version 1.0.0 — API contract for the **Production-Scale RAG on Wikipedia** course._
Every test in every `judge.json` must conform to the request body shapes below. These shapes are enforced by `cli validate`; the LLM author is constrained by them at authoring time via tool-use input schemas. **This document is the contract — there is no second source of truth.**

## Endpoints (5)

### `GET /eval`

**Run the gold-set eval and return aggregate metrics**

Triggers the agent's own eval against its bundled gold set; returns accuracy, ECE, p95 latency, cost-per-query.


`operationId`: `getEval`

**Responses**:

- `200` — Eval results

  ```json
  {
    "accuracy": 0.0,
    "ece": 0.0,
    "p95_latency_ms": 0,
    "cost_per_query_usd": 0.0,
    "n_questions": 0
  }
  ```

### `POST /feedback`

**User feedback on a citation or answer**

Capture thumbs / "missed source" / per-citation utility. Used by the agent's feedback loop to improve retrieval and synthesis over time.


`operationId`: `postFeedback`

**Request body**:

```json
{
  "trace_id": "string",
  "rating": -1,
  "per_citation": [
    {
      "citation_id": 0,

}
```

### `POST /search`

**Main RAG endpoint — answer a question with citations**

Accepts a natural-language question; returns a grounded answer with verbatim citations to Wikipedia articles, a confidence score, and a trace_id. The cited quotes MUST be verbatim substrings of their source articles — the grader verifies this deterministically.


`operationId`: `postSearch`

**Request body**:

```json
{
  "question": "string",
  "user_id": "string",
  "top_k": 0,
  "max_tokens": 0
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `question` | `string` | yes | The natural-language question. |
| `user_id` | `string` | yes | Caller identity for cost attribution. |
| `top_k` | `integer` | no | Number of chunks to retrieve. Optional, default 5. |
| `max_tokens` | `integer` | no | Synthesizer output cap. Optional. |

**Responses**:

- `200` — Successful answer

  ```json
  {
    "answer": "string",
    "citations": [
      {
        "id": 0,
        "article": "string",
        "section": "string",
        "quote": "string",
        "char_offset": 0
      }
    ],
    "confidence": 0.0,
    "trace_id": "string",
    "tokens_in": 0,
    "tokens_out": 0,
    "cost_usd": 0.0,
    "latency_ms": 0
  }
  ```
- `400` — Malformed request
- `503` — Service degraded — return when sources insufficient

### `GET /trace/{trace_id}`

**Full retrieval + reasoning trace for a query**

Returns the plan, retrieved chunks with scores, reranking decisions, the prompt sent to the model, and the model's response. The grader uses this to verify citation faithfulness and to attribute cost.


`operationId`: `getTrace`

**Responses**:

- `200` — Trace document

  ```json
  {
    "trace_id": "string",
    "spans": [
      {}
    ],
    "totals": {
      "tokens_in": 0,
      "tokens_out": 0,
      "cost_usd": 0.0,
      "duration_ms": 0
    }
  }
  ```
- `404` — Unknown trace_id
